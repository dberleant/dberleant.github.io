(mysubdomain is where w3schools is storing your sandboxes, and myfilename is the name you save the sandbox as). That way we can access it, while some other links people have sent in don't provide access. My sense is that this new w3schools sandbox saving feature is not mature yet and will be changing over time, but that's where it is currently.

However, if you prefer, just send the code in the body of your email instead (and then you don't have to make an account).

1. Make a form that contains a bunch of different features described in the W3schools lessons Form Elements,  Form input types, and Form attributes. Each feature should have a label explaining what it does. Make your action handler be https://www.w3schools.com/action_page.php and be sure the form has a "submit" input type. Add this to an existing page with other buttons already there to demonstrate things from earlier HW(s), or make a new page if you prefer.

Note: saved sandboxes do not run form action handlers properly because the path name of the action handler differs from the path of the saved sandbox area of the site. However if you paste the saved code into a regular w3schools sandbox that they provide in the lessons, it should then work fine.

2. Make a small game or demo in which you use setTimeOut() and/or clearTimeOut() along with buttons, showing how these work. Add it to your buttons page.
