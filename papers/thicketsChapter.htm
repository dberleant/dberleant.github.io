The Thickets Approach to P-Bounds

 

Daniel Berleant, 3/00

 

Monte Carlo analysis is traditionally used in risk analyses to model uncertainty in the values of inputs of various kinds, such as initial conditions and parameters. While Monte Carlo has proven useful, extensive experience has revealed limitations in the technique. These limitations motivate the development of new techniques to overcome those limitations.  These new techniques include both novel ways to apply Monte Carlo, as well as alternative approaches. This chapter focuses on an alternative approach, probability bounds (p-bounds), and on a way of implementing p-bounds, the thickets method. Next we briefly discuss limitations of Monte Carlo simulation, followed by ways of attempting to address these limitations within the Monte Carlo paradigm. Then we discuss p-bounds analysis and the thickets approach to working with p-bounds.

 

0.1 Introduction: Monte Carlo Simulation and its Limits

 

The benefits of p-bounds can be best appreciated in the context of the limitations of the traditional Monte Carlo method. That is why we begin by reviewing limitations of the Monte Carlo approach. There are two major limitations that can be significant in analyzing some of the problems we are interested in:

 

1) Lack of knowledge sufficient to specify the probability distribution function (e.g. normal, lognormal, uniform, etc.) for an uncertain input.  If some input is uncertain, then assuming it must be a particular number is certainly unjustified.  Uncertainty in value can often be modeled with a distribution function, but if insufficient information exists to determine what distribution function applies, it is impossible to draw samples as needed by Monte Carlo analysis unless either some distribution is arbitrarily assumed to apply, or the input is bounded by a range instead of characterized using a distribution function, which is a considerably weaker characterization of it.  An arbitrary distribution (e.g. the normal or “bell” curve) may be assigned to an input so that samples may be drawn from it, but while this enables a Monte Carlo analysis to proceed it does so at the cost of making an unjustified assumption about the input.  Unjustified assumptions about model inputs imply a greater or lesser degree of untrustworthiness in the results.

 

The other tactic for dealing with an uncertain input whose distribution is not known is to bound it by describing its range as an interval. Then this interval may be sampled, leading to results that bound the range of values of the outputs. However this relatively weak characterization of an input may ignore information available about the input, such as variance and mean, that could potentially be used to help characterize the outputs.

 

Examples: Kolmogorov showed that without large numbers of data points, a single PDF is likely to be significantly wrong, and that confidence limits (in the form of p-bounds) are more appropriate. Figure #3E shows an example of a distribution function and its confidence limits.

Example 1: Suppose you have encountered four hatching clutches of leatherback sea turtles, an endangered species, and counted the number of baby turtles traveling from the nest to the water in each case. From these four data points, one could still not determine the distribution function describing the distribution of numbers of baby turtles across leatherback nests.  #####revise this example as needed#####

 

Example 2: Suppose evidence suggests that the population of some endangered species (e.g. leatherback turtles, as before) is at least some minimum value but not greater than some maximum value, but that no more than that can be reasonably assumed. It would then be appropriate to specify that model input as an interval with a low bound and a high bound, but it would be inappropriate to use any particular distribution to model it.

 

2) Lack of knowledge about the precise nature of the dependency relationships among inputs.  Suppose two inputs have no significant relation to each other, just as the price of beans has no significant relation to the number of sunny days per year in Washington, D.C. in the U.S.A. If one has distribution functions for each input, each may be sampled to provide a pair of numbers to use for one run of a simulation model, without fear that the value of one sample makes the value chosen for the second input more or less likely than implied by the distribution function describing the second input.  This is good, because it makes the simulation more believable. It is also good if the value of some input completely determines the value of a second input.  For example, the number of sunny days per year completely determines the number of non-sunny days per year, and both values might be used in a simulation model of, say, utilization of tourist attractions (some of which would be more attractive on sunny days and others of which would be more attractive on non-sunny days).  Then, a sample of one input determines the sample to use of the other, and a simulation using both can proceed correctly.  However, a third possibility often occurs and presents some problems.  For example, a model of bean production might incorporate as inputs both the price of beans and the number of sunny days in bean growing areas, and it is plausible that the values of those inputs are related in some way but that one does not completely determine the other.  Unless the joint probability distribution is known (which amounts to knowing exactly what distribution to sample for one input, given what value was sampled for the other), it is not possible to know how a sample value of one input should properly be generated given a sample value of the other. 

 

Overcoming the limitations of a naïve Monte Carlo approach within the Monte Carlo paradigm.

 

Let us look at how to use Monte Carlo as best it can be used in situations in which one or more of the limitations occur. Later we will discuss how to avoid the problems inherent in this by using p-bounds.

 

In the case of a model input for which insufficient data exists to justify describing it with any particular distribution function, often a generic “reasonable” distribution is used to model it anyway, such as a normal distribution. This allows a Monte Carlo model to be completed and therefore to be run, but clearly the need to make unjustified assumptions about model inputs means that any conclusions drawn from it are less convincing and more debatable than if no such assumptions needed to be made.

In the case of a model input for which a range can reasonably be assigned but not a particular distribution function, one approach is to assign a distribution function that is guessed to be reasonable, as just described. Another approach is to model all inputs as intervals (ranges extending from minimum to maximum plausible values), even those for which a distribution function could reasonably be used. Then a Monte Carlo model could be used by picking sample points randomly within those intervals to run the model.  The results from the model will give intervals describing ranges for the outputs, but say nothing about the distributions within those intervals. Using intervals for inputs that could be modeled more precisely with distribution functions, just because some other input cannot be, means ignoring information available about those other inputs. While the conclusions drawn may be sufficient in some situations, they will tend to be weaker than they would be if information about inputs was not ignored.

In the case of lack of knowledge about the precise nature of the dependency relationships among inputs, the Monte Carlo approach may proceed if the inputs are assumed independent.  While this makes it possible to run the models, the independence assumption differs from the actual situation and can therefore lead to suspect conclusions.  The sensitivity of the conclusions to the independence assumption can be tested by also running the model under the assumption that the inputs are perfectly positively correlated, and then under the assumption that they are perfectly negatively correlated.  Thus different assumptions representing extremes will lead to different conclusions.  This process tests the sensitivity of the conclusions to the unjustified assumptions about dependency that are being made for the purpose of having a runnable model.

 

A better way is to use the degree of correlation between the two partially dependent inputs to help draw reasonable sample inputs.  This can be done if the correlation between the two inputs is known (which it may or may not be, depending on the situation).  Sometimes this correlation is known even though the full details of the dependency relationship are not.  Using this tactic, drawing a relatively high sample value for one input would increase the probability of drawing a relatively high sample value of the other input, if the correlation was positive.  Similarly, it would increase the probability of drawing a relatively low value of the other input if the correlation was negative.  This use of the correlation amounts to assuming a particular dependency relationship that, while plausible, may not be known to actually be correct.  This will tend to produce more dependable results than simply ignoring the fact that there is a dependency relationship, as is done if independence is assumed.  Nevertheless, results will still not be as trustworthy as we might wish since they are based on a dependency relationship that might not be right.

 

We have described how Monte Carlo simulation has been used in situations in which unsupported assumptions must be made in order to use it.  Ideally however, techniques would be used that do not require such assumptions.  The approach described next, probability bounds (p-bounds) analysis, is such a technique.

 

0.2 Probability Bounds and How they can Help

 

Cumulative Distributions. The probability that a sample drawn from a distribution will have a value less than or equal to the maximum value allowed by that distribution, is exactly 1.  More generally, the probability that a sample will have a value equal to or less than any given value allowed by the distribution increases if we set the given value to higher and higher numbers.   A curve generated by progressively increasing the given value and asking what the probability is of any sample being equal to or less than the given value is called the cumulative distribution function (CDF). This is illustrated in Figure QW#.

 

 

Figure QW#  –  a probability distribution function (PDF), upper graph, and its corresponding cumulative distribution function (CDF), lower graph. The CDF simply describes the cumulative area under the corresponding PDF. All CDFs rise to a final value of 1. Notice how when the PDF is high, the CDF rises more rapidly. This is because the area under the PDF from negative infinity to any given point x0 on the horizontal axis, which determines the value of the CDF at x0, cumulates more rapidly over regions of the x-axis where the value of the PDF is high.

 

What happens if we can say something about a PDF but are not able to specify it completely?  For example, perhaps we know the mean and variance but not the detailed form of the curve. In that case, there are many different PDFs that conform to the limited information we have about it. For example, almost any PDF can be shifted right or left until its mean is a given value, and then without changing its mean, stretched or compressed so as to change its variance to a given value. Such a family of curves, if many of them were drawn on the same graph, would form a jumble that would be difficult to work with. This apparent jumble can be expressed in a more visualizable way using p-bounds.

P-bounds. If we convert a family of PDFs into a different format, namely the equivalent family of CDFs, it is considerably easier to visualize and work with. Figure DS# shows the PDF family consisting of all PDFs with a given mean and variance, expressed in the form of a family of CDFs. This family is described by bounds within which all CDFs in the family lie. Those two bounding curves are the p-bounds.

 

Figure DS# - P-bounds for the family of all CDFs with mean 10 and variance 5. All such CDFs fall within these bounds (however, the bounds themselves do not have mean 10 and variance 5). Each bounding curve has a long “tail,” which gradually becomes steep as it nears the mean of 10, and then its slope becomes more gentle again in a small region as it gets even nearer to 10. After passing 10 (to the right in the case of the top curve and to the left in the case of the bottom curve), the curves are flat.

 

As you can see, p-bounds provide easily visible bounds on the space through which CDFs in the family can travel.  These bounds are called probability bounds or p-bounds because they bound the space of CDFs (and therefore implicitly bound the less easily visualized space of PDFs).

P-bounds can be used to draw some interesting conclusions.  For example, suppose the mean and variance are known for the distribution of grades in a medium sized class of students who took a course on mathematical ecology last semester, while at the same time the actual distribution was quite uneven and we do not believe that the distribution this semester will reflect last semester’s uneven actual distribution.  What is the expected percentage of A’s this semester?

 

To answer this question, note that the mean and variance from the previous semester are more reliable predictors of the mean and variance this semester than the detailed distribution from the previous semester is a predictor of the detailed distribution this semester. This is because the detailed distribution is apt to have unevennesses due to randomness in the pool of students who took the class last time. Consider the p-bounds conforming to the mean and variance, shown in figure #M. Assume that a grade of 90% or better results in an “A.” The lower p-bound shows a probability of 67%##### that a given student’s grade will be below 90%, and therefore a 33% probability that a student’s grade will be 90% or above. The upper p-bound shows a 75% chance that a grade will be below 90% and therefore a 25% probability that it will be 90% or above.  Then we conclude that the probability of getting an “A” is within the interval [0.25, 0.33], which implies that the expected frequency of “A”s is between 25% and 33%.

Combining two uncertain inputs. Next, consider the case of a model with two uncertain inputs that must be combined. As an example of such a model, many bat populations have suffered declines in population in recent years, due to such factors as pesticides in their diet of insects  and human disturbance of habitats. In order to estimate the number of bats of a particular species in a particular area that will be present one year from now, one can multiply the current population by the growth rate (a negative growth rate would signify a continued decline in population). Both of these figures, the current population and the growth rate, are unlikely to be known with any great accuracy, and therefore might be better modeled using distributions than point values. Thus we would have to multiply two distributions together to get an estimate of the population in a year. Furthermore, the dependency relationship between these two distributions is unknown. They could be completely independent (which would lead to eventual extinction if the growth rate stayed negative for long enough). Alternatively, it is possible that they are positively correlated to a greater or lesser degree. This can occur in populations that are sufficiently low to be marginally viable. In that case an increase in population can lead to an increase in growth rate. Finally, it is possible for population and growth rate to be negatively correlated. This can occur when a population approaches the limit of the ability of the environment to support it, at which point individuals are forced into competition with each other for food and perhaps other resources, making it harder for them to survive and reproduce.

 

Thus, population and growth rate each have some specified distribution function, but the dependency relationship between the two distribution functions may not be known. What can be said about the product of population and growth rate (that is, the prediction for population in a year)? It is not a single figure, because the inputs to the product operation are not precise numbers but rather they are probability density functions representing partial knowledge about the true number. If we could assume those distributions were independent (the value of one does not affect the value of the other) then the result of multiplying them would be a distribution. Or, if we could assume those distributions were completely correlated (that is, a higher value for one input implies a correspondingly higher value for the other) or negatively correlated (a higher value for one input implies a correspondingly lower value for the other) then the result would again be some distinct distribution. To exemplify this idea, Figure #####R shows the different cumulative distributions resulting from multiplying two input distributions in the three cases where their dependency relationship is independent, correlated completely and positively, and correlated completely and negatively.

 

Figure #R. The distributions for two random variables ((a) and (b)) and for their product under three different dependency conditions, independent (c), fully positively correlated (d), and fully negatively correlated (e). The numbers on the x-axis are illustrative of the general case, and would differ for any particular bat or other population.

 

Since we cannot justify any particular dependency relationship in such a case, the result could be any of a family of distributions, each one corresponding to some dependency relationship - whether simple or complex - between the input distributions. Then,  the family of all possible result distributions (which includes independence, full positive and negative correlation, and all other dependency relationships) may be expressed using p-bounds to represent the space within which each member of that infinitely numerous family must be. Figure #####S shows an example of this.

When one input is a distribution but the other’s distribution is unknown.

 

Now, let us consider the case where one input is described using a distribution function and another is less well characterized, being described only by an interval describing its range of plausible values. The presence of an input described by an interval prevents describing an output or other value representing the sum, product, etc. of the input interval and the input distribution function, as a distribution function.  As Figure #T shows, each possible value of the interval, when combined with the distribution function, leads to a distinct distribution for the output variable. The result is a family of distributions that are possible for the output. This family may be described using p-bounds.

 

Figure #T. Two example values, when multiplied by a distribution function, lead to two different distribution functions describing the result (a). When multiplying a distribution function by an interval, each value within the interval leads to a different result distribution function (b). Only a few are actually shown, although since the interval holds an infinite number of values, an infinite number of distributions are needed to describe the output. The family of plausible distribution functions for the output may be converted into their cumulative forms, leading to a family of cumulative distributions that all lie between top and bottom bounding cumulative distributions (shown in bold), which are the p-bounds (c).

 

As  noted earlier the traditional Monte Carlo approach can require assumptions that are unsupported by available evidence. This can lead to results that are not as trustworthy as we might wish, because of the possibility that the obtained results are significantly affected by these assumptions. Unsupported assumptions reduce the dependability of resulting conclusions.

 

P-bounds form an approach which can avoid the need for unsupported assumptions, improving the dependability of resulting conclusions. In the previous discussion p-bounds were illustrated - literally - with drawings. But computer software for working with p-bounds needs to represent them using numbers. In the next section, one method for calculating with p-bounds that can be implemented in computer software is described.

 

0.3 Thickets: an Approach to Working With P-Bounds

 

A suitable way to work with p-bounds on computers is based around intervals and associated probability masses. An interval, for our purposes, is simply a range described by low and high end points. For example, the interval containing all numbers from 2 through 9.8  is written [2, 9.8]. In the thicket method for working with p-bounds, each interval is associated with a probability, which assigns probability to the event that a value of the random variable falls within that interval. If another interval with its own associated probability mass overlaps the interval, the region of overlap has two probability masses affecting with it. Figure #V shows examples of symbolic descriptions, their “thicket” visualizations (so-called due to the “thicket”-like collection of tall rectangles on the horizontal axis), and the resultant p-bounds.

The thicket visualizations show the symbolic descriptions in graphic form, albeit not perfectly since the flat tops of the rectangles are a graphical artifact and do not imply flat distributions (in fact, the form of the distribution of probability mass within a given range is undefined).  The p-bounds (computed) column shows the cumulative distributions corresponding to the thicket visualizations and symbolic descriptions.

 

You may be wondering about the “stair step” shape of the curves in the “p-bounds (computed)” column of Figure #V. This is due to the way the p-bounds are computed from the symbolic descriptions. The top p-bound is the fastest-rising curve that is consistent with the symbolic description, and is computed by asking what the cumulative form of the distribution function looks like if the probability mass associated with each interval is concentrated at the low bound of the interval. Thus if the interval is [1,2] and the probability mass is .01, then the cumulative distribution jumps up by .01 as soon as the value on the horizontal axis reaches 1, because the mass of .01 is taken to be concentrated at the low bound of the interval. Each interval contributes its own upward jump, the size of which is determined by the probability mass associated with that interval, and the x-axis location of which is determined by the low bound of the interval. To get the bottom p-bound, the probability masses are concentrated at the high bounds of their intervals instead of the low bounds. Note that if the probability mass was not concentrated at any location in an interval but was spread evenly throughout, the cumulation would rise smoothly. See Figure #W.

 

Figure #W. If the probability mass associated with an interval is concentrated at the interval’s low bound, the cumulative curve rises suddenly at the low bound of the interval (top curve). If on the other hand the probability mass is concentrated at the interval’s high bound, the cumulative curve rises suddenly at the high bound of the interval (bottom curve). However, if the probability mass is distributed evenly over the interval, the cumulative curve rises gradually at a constant slope (middle curve).

 

The top and bottom curves bound the family of curves that could result from any distribution of probability masses within their associated intervals. Therefore distribution of masses within their intervals can lead to a cumulative curve that either curve. Thus the top and bottom curves are conservative in that they represent the most extreme cumulative distributions possible given the symbolic description.

 

The stair step shaped bounding curves become closer and closer to smooth curves as the sizes of the steps get smaller. At the same time, the symbolic descriptions get longer because they contain more (and narrower) intervals with smaller masses. The process of representing a smooth curve using steps is called discretization. A reason for using larger steps is simply to allow computers to process the symbolic descriptions faster. Computers can calculate very fast compared to human beings, but computation tasks can sometimes overwhelm even a computer’s speed and cause it to produce results relatively slowly (as evidence of this, consider that if computers could compute nearly instantaneously, users would experience fewer delays while running programs).

 

0.4 Computing With Thickets: an Example

 

At this point we have introduced the thickets representation for p-bounds. However, we have not yet shown how to take two different variables, each represented by p-bounds, and add, subtract, multiply, or divide them. We introduce how to do these computations via an example.

 

The example: suppose we are interested in the total amount of some pesticide released into the environment, which we model as the amount contributed by U.S. agriculture (call it “A”) plus the amount contributed by all other countries (call it “B”). Assume we do not know the exact amount of either A or B, but we can describe these amounts using distributions. (Probably the curve describing B, the amount contributed by all other countries, will be broader and shallower than the curve for A because our knowledge about its value is weaker.) We can discretize the distribution function for each of them by describing them with thickets. Note that inside the computer, thickets would be represented symbolically, but we will think about them using the thicket visualization as needed because this can make them easier to think about.  Figure #W shows the thicket visualization and numerical description for this situation.

0.5 Finding Points on the P-Bounds With Linear Programming

 

To find the value of either the top or bottom p-bound at any given value on the horizontal axis, we must maximize or minimize the sum of the probability values of some subset of the interior cells in a table, such as we have been discussing, which describes the joint probability distribution of two random variables. This table constrains how we can rearrange the probabilities of the interior cells as we seek to maximize or minimize. In particular, the marginal probabilities impose a value on the sum of the probabilities of the cells in each column, and a value on the sum of the probabilities of the cells in each column. These values are given and cannot be changed. In Figure #CB these probability values are shown in bold. Thus, the task is to maximize or minimize a sum of values given other sums of values that are fixed. This type of situation lends itself to a technique called linear programming. Linear programming is a technique widely used in diverse areas of science and engineering, and fortunately there are numerous software packages available for solving a linear programming problems, both commercial and public domain. Computer  program listings are also widely printed in books (e.g. [citationR]). We will not describe how to write a computer program for doing linear programming, but instead show how to set up a linear programming problem whose solution would be one of the maximized values we seek. Each such maximization problem presents a similar but distinct linear programming problem. (To minimize the sum of a subset of interior cells, we can simply maximize the sum of the others and subtract that value from 1.0.)

0.6 Converting P-Bounds to Thickets

 

We have now finished outlining how to calculate the p-bounds for the result of an arithmetic operation on two thickets: portray the thickets as a table containing marginals and interior cells, generate an initial plausible set probabilities for the interior cells, and then rearrange probabilities among those interior cells using linear programming to get a point on the top or bottom p-bound. This rearranging process is be done for each various points on the p-bounds so that those points can be connected with line segments to produce the full p-bounds.

 

Thus, we begin with thickets and end up with p-bounds. What if we wish to use the p-bounds as the input to another calculation? For example, we add two thickets, X and Y to produce p-bounds for their sum, Z=X+Y. Now we wish to multiply (say) Z*W where W is a thicket. To do this we need to be able to convert the p-bounds for Z back into a thicket. Here is a procedure for doing this.

 

1)      To start with, the p-bounds to be converted to thickets must have both the top and bottom curves in stair step form – that is, each curve is a sequence of horizontal and vertical line segments attached end-to-end. If the p-bounds contain diagonal or curving portions, they must be converted into stair step form. To do this safely requires defining stair step curves that are outside the p-bounds. The newly defined curves constitute new p-bounds to use as a starting point for subsequent steps. This is illustrated in Figure ZS#.
